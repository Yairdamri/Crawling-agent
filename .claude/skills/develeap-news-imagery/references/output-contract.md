# Output contract — the decision object

This is the schema the skill produces, per article, regardless of which tier won. Share this file with the consuming dev when they ask "what does the JSON look like?".

The contract is intentionally surface-agnostic. The same decision object can drive a WordPress plugin, a Figma mockup, a social card, or an internal dashboard — none of them touch the original `data/news.json`.

## Top-level shape

```json
{
  "article_id": "<URL — primary key>",
  "url":        "https://...",
  "title":      "...",
  "source":     "...",
  "category":   "AI | DevOps | Cloud | Engineering | Security | Other",
  "tags":       ["..."],

  "status":     "ok | failed",
  "error":      null,

  "decision":   { ... see below ... },

  "rejected_sources": [
    { "tier": 1, "reason": "license unclear on vendor press page" },
    { "tier": 2, "reason": "no event photo with reusable license" }
  ]
}
```

A manifest is a JSON array of these objects.

## Field reference

### Identity & passthrough

| Field | Type | Required | Notes |
|---|---|---|---|
| `article_id` | string | yes | The article URL. Primary key. Same value the Crawling-agent uses. |
| `url` | string | yes | Same as `article_id`. Kept separately so consumers don't have to assume. |
| `title` | string | yes | Passed through from `news.json`. |
| `source` | string | yes | RSS source name (e.g. `"The Hacker News"`). For attribution context only. |
| `category` | enum | yes | One of `AI / DevOps / Cloud / Engineering / Security / Other`. |
| `tags` | string[] | yes | Passed through. May be empty. |

### Status

| Field | Type | Required | Notes |
|---|---|---|---|
| `status` | enum | yes | `"ok"` or `"failed"`. |
| `error` | string \| null | yes | Human-readable error string when `status = "failed"`. `null` otherwise. |

When `status = "failed"`, `decision` may still be partial (e.g. `prompt_used` is set, `image_path` is `null`). Consumers should check `status` first.

### `decision` object

| Field | Type | Required | Notes |
|---|---|---|---|
| `source_type` | enum | yes | One of: `"ai_pollinations_flux"`, `"press_image"`, `"event_photo"`, `"brand_logo_bg"`. |
| `image_url` | string \| null | yes | Source URL for Tier 1/2 (do **not** re-host). `null` for Tier 3/4. |
| `image_path` | string \| null | yes | Local file path for Tier 4 (and Tier 1/2 if the consumer chose to download). `null` for Tier 3. |
| `image_hash` | string \| null | optional | sha256 of the image bytes. Set by the consumer if it uploads to a CDN; the skill leaves it `null`. |
| `prompt_used` | string \| null | yes (Tier 4) | Full prompt sent to Pollinations (subject + locked suffix). `null` for Tier 1-3. |
| `lead_color` | string \| null | yes (Tier 3/4) | Hex color, e.g. `"#326CE5"`. Drives the background tint or the AI subject sentence's `...on a [color] backdrop`. |
| `composition` | enum | optional | `"isometric"` is the only currently supported value. Reserved for future variation. |
| `license_note` | string | yes | Verbatim license text (Tier 1/2) or fixed boilerplate for Tier 4 (`"Generated by Pollinations.ai (Flux model). Develeap holds usage rights."`). |
| `attribution` | string \| null | yes (Tier 1/2/3) | Credit line to display on the surface. `null` for Tier 4. |
| `alt_text` | string | yes | One factual sentence describing the frame. Required for accessibility regardless of tier. |

### `rejected_sources` array

Optional, but strongly recommended when the skill walked tiers and rejected some. Lets the consumer audit decisions later without rerunning. Each entry:

```json
{ "tier": 1, "reason": "<short string>" }
```

Empty array (or omitted) when the skill didn't walk the priority list — e.g. a forced Tier 4 batch.

### Failure mode

```json
{
  "article_id": "https://example.com/some-article",
  "url": "https://example.com/some-article",
  "title": "...",
  "source": "...",
  "category": "Other",
  "tags": [],
  "status": "failed",
  "error": "Pollinations: exhausted retries: HTTP 429",
  "decision": {
    "source_type": "ai_pollinations_flux",
    "image_url": null,
    "image_path": null,
    "image_hash": null,
    "prompt_used": "<full prompt>",
    "lead_color": "#1A2A44",
    "composition": "isometric",
    "license_note": "Generated by Pollinations.ai (Flux model). Develeap holds usage rights.",
    "attribution": null,
    "alt_text": null
  },
  "rejected_sources": []
}
```

When a consumer sees `status = "failed"`, it should leave the article without a cover image and let the next pipeline run retry — no "best-effort" stand-in image and no schema gymnastics.

## Stability and versioning

- Field names in `decision` are stable. Adding a new optional field is backwards-compatible; renaming or removing one is not.
- New `source_type` values may be added later (e.g. `"ai_mystic"` if we revisit Mystic). Consumers should fail soft on unknown values, not crash.
- The contract is **separate** from `news.json`'s `schemaVersion`. The skill never bumps `schemaVersion`; the dev does, when (and if) folding `decision` data back into the article schema.
