# Develeap news‑feed imagery — image picker

## What this skill does, and what it doesn't

**Does:** for each news article (typically an item from Crawling‑agent's `data/news.json`), decide which image to use and produce it. Output is a portable decision per article — image local path, plus the provenance that lets the consumer trust and credit it.

**Doesn't:** modify any surface, repo, schema, or pipeline. The skill produces image files at predictable paths. The dev (Yair) decides how to wire them into the schema and the WordPress plugin.

The point of this scoping: keep the rules and the generation pipeline reusable across whatever surfaces Develeap ships news on (Figma mockups, the live website, social posts, an internal dashboard). Surface integration is the consumer's job.

## When this skill runs in production

The expected use is **per article** as Crawling‑agent emits new items. Read `references/integration.md` once for the full pipeline picture. The condensed version:

- **Source of truth:** `data/news.json` in the Crawling‑agent repo. Schema version 1.
- **Article shape:** `{ url, title, summary, source, tags[], score, category, publishedAt }` — see `references/integration.md` for the exact fields.
- **Filename convention:** `sha256(article.url)[:16].jpg` — matches Crawling‑agent's existing PoC at `scripts/test-image-gen.mjs`. The script in this skill emits exactly this filename when given a `url` field.
- **Output directory:** default to `data/images/` relative to the consumer's repo root. The consumer points the script's `--out-dir` there.
- **Hard constraint:** the skill **must not** modify anything in the Crawling‑agent repo. Produce image files only. The dev decides how/when to add an `image` field to the schema and bump `schemaVersion`.

## Output contract

For every article you process, return a `decision` object with this shape (full schema in `references/output-contract.md`):

```json
{
  "article_id": "<URL — primary key>",
  "url":        "https://...",
  "title":      "...",
  "source":     "...",
  "category":   "AI | DevOps | Cloud | Engineering | Security | Other",
  "tags":       ["..."],
  "status":     "ok | failed",
  "error":      null,
  "decision": {
    "source_type":  "ai_pollinations_flux | press_image | event_photo | brand_logo_bg",
    "image_url":    null,                  // only for press_image / event_photo
    "image_path":   "/abs/data/images/<sha256_16>.jpg",
    "image_hash":   null,                  // optional, set by consumer if uploaded elsewhere
    "prompt_used":  "<subject>, <STYLE_SUFFIX>",
    "lead_color":   "#D97757",
    "composition":  "isometric",
    "license_note": "Generated by Pollinations.ai (Flux model)...",
    "attribution":  null,
    "alt_text":     "..."
  },
  "rejected_sources": []
}
```

## Decision workflow per article

For each article, walk these steps:

1. **Pick the source tier.** Walk the priority list in `references/source-priority.md`. Use the first tier that passes the legality test. Record any tiers you checked and rejected.
2. **Pick the lead color.** Use `references/brand-colors.md`: subject's brand color (lead or accent) + Develeap palette. Honor the page‑level anti‑uniformity guards if you're processing a batch.
3. **Compose the subject sentence.** Use the patterns in `references/prompt-recipes.md`. **Then apply `references/composition-discipline.md` BEFORE submitting** — Flux is looser than Mystic, and the discipline rules are what keep the magazine look sharp.
4. **Produce the image.**
   - Tier 1 (press image) / Tier 2 (event photo): record the source URL, attribution, and license note. Don't re‑host.
   - Tier 3 (brand logo on background): record the brand color, the logo URL, and the layout intent. Don't generate a composite — that's the consumer's job.
   - Tier 4 (AI generation via Pollinations): pass the article record to `scripts/generate_images.py`. The script downloads the image to `<out-dir>/<sha256_16>.jpg` and returns a manifest entry.
5. **Write a short alt text.** One factual sentence describing what's in the frame.
6. **Self‑audit (batch only).** If you processed ≥4 articles, scan the chosen lead colors and compositions against the anti‑uniformity guards. Regenerate just the offenders.

## The locked Pollinations recipe

The image generator is **Pollinations.ai (Flux model)**. Free, no API key, returns image bytes from a single GET — no async polling.

```
GET https://image.pollinations.ai/prompt/{URL_ENCODED_PROMPT}
    ?width=1376&height=768&model=flux&seed={N}
    &nologo=true&private=true&enhance=false
```

- `width=1376&height=768` → 16:9, matches the news‑feed image slots.
- `model=flux` → quality. `turbo` and other models drift away from the magazine look.
- `private=true` keeps the image off the Pollinations public feed.
- `enhance=false` because we shape the prompt ourselves.

The `STYLE_SUFFIX` is encoded in `scripts/generate_images.py`. It already includes the composition guards (centered, generous negative space, no text, 3D isometric perspective) — but you still need to write a clean **subject sentence** per `references/composition-discipline.md`. Note: clouds are **not** in the suffix; add them in the subject sentence only when they strengthen the metaphor (cloud platforms, observability, empty frames). Default is no clouds.

**Do not put any color in the suffix.** Color comes from the per‑article subject sentence — that's how you get variety across a batch.

**Rate limit.** Anonymous Pollinations requests are throttled to ~1 concurrent per IP. The script paces itself with a 3 s inter‑request delay and exponential backoff on 429. For a 17‑image page expect ~5–10 minutes of wall time.

## Inputs the script accepts

`scripts/generate_images.py` takes a JSON list. Two accepted shapes:

**Shape A** (preferred — matches Crawling‑agent's article object):

```json
[
  {
    "url":      "https://thehackernews.com/2026/05/...",
    "title":    "CISA Adds ...",
    "source":   "The Hacker News",
    "tags":     ["CVE", "Linux"],
    "category": "Security",
    "subject":  "<full subject sentence per prompt-recipes.md + composition-discipline.md>",
    "lead_color":  "#A4262C",
    "composition": "isometric"
  }
]
```

When `url` is given, the output filename is `sha256(url)[:16].jpg` — drop‑in compatible with the Crawling‑agent convention.

**Shape B** (legacy, for ad‑hoc batches not tied to a URL):

```json
[ { "article_id": "any string", "subject": "..." } ]
```

## Don't surprise the user

- **Free, but quota‑guarded.** Pollinations Flux is free, but the public endpoint is rate‑limited. For very large batches (>50 articles in one run), warn the user before kicking off — it'll take ~30+ minutes.
- **Legality is non‑negotiable.** Never substitute a "probably fine" real image. If a Tier 1‑3 source can't be cleared 100 %, drop down to Tier 4 — the Pollinations image is owned by Develeap and safe.
- **Don't compose Tier 3 yourself.** Even though the skill knows the layout intent (logo + brand‑tinted background), composing is the consumer's responsibility — they may want it as SVG / React / Figma / static PNG depending on surface.
- **Don't touch the Crawling‑agent repo.** The skill is downstream of `data/news.json`, never upstream of it. See `references/integration.md` for the hard list of things the skill must not do.
